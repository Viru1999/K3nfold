01 Propositional Logic Foundation
What is a Proposition?
A statement that is either TRUE or FALSE. Not a question, not an opinion, not "x > 2" by itself (that's open — depends on x).
✓ "17 is prime" → TRUE proposition
✓ "Every even number > 2 is a sum of two primes" → proposition (Goldbach's conjecture — unknown!)
✗ "Is 5 odd?" → not a proposition (question)
✗ "x + 1 = 3" → not a proposition until x is fixed (it's a predicate)
Connectives — Combining Propositions
You build complex statements from simple ones using connectives.

AND (∧) — P ∧ Q is TRUE only when both P and Q are true.
OR (∨) — P ∨ Q is TRUE when at least one is true. (Inclusive or — both allowed.)
NOT (¬) — ¬P flips the truth value.
IF...THEN (→) — P → Q. The ONLY time this is FALSE is when P is TRUE but Q is FALSE.
IFF (↔) — P ↔ Q. TRUE when both have the same truth value. "P if and only if Q."
⚠ The implication P → Q is TRUE when P is FALSE, regardless of Q. This confuses everyone. "If pigs fly, I'll give you £1M" — since pigs don't fly, the statement is vacuously true. In analysis, this appears in definitions constantly.
Negation Rules — How to Negate Complex Statements
Negating correctly is half of proof writing. Memorise these.

¬(P ∧ Q) ≡ ¬P ∨ ¬Q   [De Morgan]
¬(P ∨ Q) ≡ ¬P ∧ ¬Q   [De Morgan]
¬(P → Q) ≡ P ∧ ¬Q     [P is true but Q fails]
¬(P ↔ Q) ≡ P ↔ ¬Q    [they differ]
Example: Negate "If n is even then n² is even"
¬(P → Q) = P ∧ ¬Q = "n is even AND n² is NOT even" — this is what you assume in proof by contradiction.
Contrapositive & Converse — Know the Difference
Given P → Q:

Contrapositive: ¬Q → ¬P — logically EQUIVALENT to P → Q. Proving the contrapositive proves the original.
Converse: Q → P — NOT equivalent. Completely different claim. Confusing these is a classic error.

P ↔ Q means proving BOTH P → Q AND Q → P (equivalently, P → Q and contrapositive Q → P fails... no: prove both directions).
P: "n is divisible by 4"   Q: "n is even"
P → Q: TRUE. If divisible by 4, then even. ✓
Converse Q → P: FALSE. 6 is even but not divisible by 4. ✗
Contrapositive ¬Q → ¬P: "if n is odd, then n is not divisible by 4." TRUE (same as P→Q). ✓
Quantifiers — ∀ and ∃
These are the language of mathematics. Every theorem in Real Analysis uses them.

∀ x ∈ S, P(x) — "For ALL x in S, P(x) is true." To PROVE: show P(x) for an arbitrary x. To DISPROVE: find one x where P(x) fails (a counterexample).

∃ x ∈ S, P(x) — "There EXISTS at least one x in S with P(x) true." To PROVE: exhibit a specific x. To DISPROVE: show no such x exists.
ε-N definition of limit (you will see this soon):
∀ ε > 0, ∃ N ∈ ℕ, ∀ n > N, |aₙ − L| < ε
Read: "For every epsilon (however small), I can find a point N beyond which all terms are within epsilon of L."
The ORDER of quantifiers matters enormously — ∀∃ ≠ ∃∀.
⚠ Negating quantifiers: ¬(∀x, P(x)) = ∃x, ¬P(x)  |  ¬(∃x, P(x)) = ∀x, ¬P(x)
You will use this every time you do a proof by contradiction in analysis.

03 Proof Methods Core Skill
Direct Proof
  • Assume the hypothesis P is true
  • Use definitions, algebra, logic
  • Derive Q step by step
  • Conclude: therefore Q is true □
Use when: the path from P to Q is straightforward.
Proof by Contrapositive
  • To prove P → Q, instead prove ¬Q → ¬P
  • Assume ¬Q (conclusion is false)
  • Show this forces ¬P (hypothesis fails)
  • Logically identical to P → Q □
Use when: assuming ¬Q gives you more to work with.
Proof by Contradiction
  • Assume the OPPOSITE of what you want to prove
  • Reason carefully from this assumption
  • Derive something impossible (0 = 1, x < x, etc.)
  • Conclude: assumption was wrong □
Use when: proving existence, irrationality, or uniqueness.
Proof by Cases
  • Split into exhaustive cases (cover all possibilities)
  • Prove the result in each case separately
  • All cases must be covered with no gaps
  • Result holds in all cases □
Use when: "n is even or odd", absolute values, max/min.
★ Classic Example: √2 is Irrational (Contradiction)
Claim: √2 ∉ ℚ

Assume for contradiction that √2 ∈ ℚ. Then √2 = p/q where p, q ∈ ℤ, q ≠ 0, and gcd(p,q) = 1 (fully reduced).

Then 2 = p²/q², so p² = 2q². Thus p² is even, so p is even (if p were odd, p² would be odd).

Write p = 2k. Then (2k)² = 2q², so 4k² = 2q², so q² = 2k². Thus q is also even.

But then both p and q are even → gcd(p,q) ≥ 2. Contradiction with gcd(p,q) = 1. □

Note every step: the key was choosing gcd = 1 at the start, which is the contradiction target. This is how you plan contradiction proofs.
★ Classic Example: If n² is Even then n is Even (Contrapositive)
Claim: ∀n ∈ ℤ, n² even → n even.

Contrapositive: n odd → n² odd.

Assume n is odd. Then n = 2k + 1 for some k ∈ ℤ.
n² = (2k+1)² = 4k² + 4k + 1 = 2(2k² + 2k) + 1, which is odd. □

This is cleaner than direct proof. Choosing contrapositive over direct is a key skill.
04 Set Theory Language of Math
Operation Definition Key Fact
A ∪ B x ∈ A ∪ B ↔ x ∈ A ∨ x ∈ B Everything in either set
A ∩ B x ∈ A ∩ B ↔ x ∈ A ∧ x ∈ B Only what's in both
A \ B x ∈ A \ B ↔ x ∈ A ∧ x ∉ B A minus elements of B
Aᶜ x ∈ Aᶜ ↔ x ∉ A Everything NOT in A (w.r.t. universe)

How to Prove A ⊆ B (and A = B)
To prove A ⊆ B: Take an arbitrary x ∈ A. Show x ∈ B. Done.
This is the standard method. Never try to argue globally — always use "Let x ∈ A..."

To prove A = B: Prove A ⊆ B AND B ⊆ A. Two separate arguments.
Structure: "⊆: Let x ∈ A... [show x ∈ B]. ⊇: Let x ∈ B... [show x ∈ A]."
Prove A ∩ (B ∪ C) = (A ∩ B) ∪ (A ∩ C)
⊆: Let x ∈ A ∩ (B ∪ C). Then x ∈ A and (x ∈ B or x ∈ C).
Case 1: x ∈ B → x ∈ A ∩ B → x ∈ (A∩B)∪(A∩C). ✓
Case 2: x ∈ C → x ∈ A ∩ C → x ∈ (A∩B)∪(A∩C). ✓
⊇: Similar. (Try yourself.)
De Morgan's Laws for Sets
(A ∪ B)ᶜ = Aᶜ ∩ Bᶜ     "Not (A or B)" = "Not A, and Not B"
(A ∩ B)᷊ = Aᶜ ∪ Bᶜ     "Not (A and B)" = "Not A, or Not B"

These mirror the logical De Morgan laws. In analysis you use these when taking complements of open/closed sets, and in Measure Theory constantly.
Indexed Families of Sets
⋃ᵢ∈I Aᵢ = {x : ∃i ∈ I, x ∈ Aᵢ}     x is in at least one Aᵢ
⋂ᵢ∈I Aᵢ = {x : ∀i ∈ I, x ∈ Aᵢ}     x is in every Aᵢ

In Real Analysis: open sets are unions of open intervals. The intersection of infinitely many open sets may not be open — this distinction matters enormously.
⋂ₙ₌₁^∞ (−1/n, 1/n) = {0} — intersection of open sets is a single point (closed!). This is a key counterexample.
05 Relations Order & Equivalence
What is a Relation?
A relation R on set A is a subset of A × A. We write aRb to mean (a,b) ∈ R.
Examples: ≤ on ℝ, divisibility on ℤ, "same remainder mod 3" on ℕ.
Reflexive
∀a ∈ A, aRa. Every element is related to itself.
✓ ≤ on ℝ (x ≤ x). ✓ "same parity" (n and n always have same parity). ✗ strict < (x < x is false).
Symmetric
∀a,b ∈ A, aRb → bRa. If a relates to b, then b relates to a.
✓ "same parity". ✗ ≤ (3 ≤ 5 but not 5 ≤ 3). ✗ divisibility (2|6 but not 6|2).
Transitive
∀a,b,c ∈ A, aRb ∧ bRc → aRc. Chains collapse.
✓ ≤ on ℝ. ✓ divisibility. ✓ subset ⊆. ✓ "same parity".
Equivalence Relation = Reflexive + Symmetric + Transitive
Partitions the set into equivalence classes [a] = {b : aRb}.
Key examples: congruence mod n on ℤ. "Same cardinality" on sets.
In analysis: equivalence classes appear in Lᵖ spaces (functions equal a.e. are "the same"). Foundation of MAT4023 Measure Theory.
Partial Order = Reflexive + Antisymmetric + Transitive
Antisymmetric: aRb ∧ bRa → a = b. (If both ≤, then equal.)
Examples: ≤ on ℝ, ⊆ on power sets, divisibility on ℕ.
Total order: additionally every pair is comparable (a ≤ b or b ≤ a). ℝ with ≤ is a total order.
Supremum and infimum in MAT1023 live in this framework.
06 Functions Maps & Structure
Formal Definition
A function f : A → B is a relation where every element of A is paired with exactly one element of B.
A is the domain. B is the codomain. The range/image is f(A) = {f(x) : x ∈ A} ⊆ B.

Important: range ⊆ codomain, but they need not be equal. This distinction matters for surjectivity.
Injective (One-to-One)
f(x) = f(y) → x = y. Different inputs give different outputs. No collisions.
Equivalently: x ≠ y → f(x) ≠ f(y).

To prove injective: Assume f(x) = f(y), then show x = y.
To disprove: Find x ≠ y with f(x) = f(y).
f : ℝ → ℝ, f(x) = 2x + 3. Suppose f(x) = f(y): 2x+3 = 2y+3 → x = y. ✓ Injective.
g(x) = x². Not injective: g(2) = g(−2) = 4 but 2 ≠ −2.
Surjective (Onto)
∀b ∈ B, ∃a ∈ A such that f(a) = b. Every output value is achieved.
The range equals the codomain: f(A) = B.

To prove surjective: Take arbitrary b ∈ B, explicitly find/construct a ∈ A with f(a) = b.
f : ℝ → ℝ, f(x) = 2x + 3. Given b ∈ ℝ, set a = (b−3)/2. Then f(a) = b. ✓ Surjective.
g : ℝ → ℝ, g(x) = x². Not surjective: no real x with x² = −1.
Bijective = Injective + Surjective
A perfect pairing. Every b ∈ B corresponds to exactly one a ∈ A. Invertible.
A bijection f : A → B means A and B have the same "size" (cardinality).

This is how we prove |ℕ| = |ℤ| = |ℚ| (countably infinite) and |ℝ| > |ℕ| (uncountable).
⚠ The existence of a bijection ℕ → ℚ (rational numbers are countable) is a beautiful result you will prove in MAT1023. The non-existence of any bijection ℕ → ℝ (Cantor's diagonal argument) shows ℝ is a "larger" infinity.
Composition & Inverse
Composition (g ∘ f)(x) = g(f(x)). Apply f first, then g. Order matters: g ∘ f ≠ f ∘ g in general.

Inverse f⁻¹ exists ↔ f is bijective. f⁻¹(b) = the unique a with f(a) = b.

Key facts: (g ∘ f)⁻¹ = f⁻¹ ∘ g⁻¹ (reverse order). f ∘ f⁻¹ = id_B. f⁻¹ ∘ f = id_A.
Image & Preimage of Sets
f(S) = {f(x) : x ∈ S} — image of S under f (what S maps to)
f⁻¹(T) = {x ∈ A : f(x) ∈ T} — preimage of T (note: f⁻¹ here is NOT the inverse function — it works even when f is not bijective)

Key properties (used constantly in topology and analysis):
f⁻¹(T₁ ∪ T₂) = f⁻¹(T₁) ∪ f⁻¹(T₂)
f⁻¹(T₁ ∩ T₂) = f⁻¹(T₁) ∩ f⁻¹(T₂)
f⁻¹(Tᶜ) = (f⁻¹(T))ᶜ
These DO NOT all hold for f(S) — the image is less well-behaved. In analysis: continuous functions are defined by preimages of open sets being open (MAT3133).
07 Mathematical Induction Prove for All ℕ
State what you're proving Write P(n) clearly. E.g. "P(n): 1 + 2 + ··· + n = n(n+1)/2 for all n ∈ ℕ."
Base Case Prove P(1) (or P(0)) directly. Just substitute and verify. This grounds the chain.
Inductive Hypothesis Assume P(k) is true for some arbitrary fixed k ≥ 1. Write it out explicitly — you WILL use it.
Inductive Step Prove P(k+1) using the inductive hypothesis. The key move: relate the (k+1) case back to the k case, then apply the hypothesis.
Conclude "By induction, P(n) holds for all n ∈ ℕ." □
★ Worked Example: Sum Formula
Claim: ∀n ∈ ℕ, 1 + 2 + ··· + n = n(n+1)/2

Base: n=1: LHS = 1, RHS = 1·2/2 = 1. ✓

Hypothesis: Assume 1+2+···+k = k(k+1)/2 for some k ≥ 1.

Step: Show it for k+1:
1+2+···+k+(k+1) = k(k+1)/2 + (k+1)  [by hypothesis]
= (k+1)[k/2 + 1] = (k+1)(k+2)/2 = (k+1)((k+1)+1)/2 ✓

Conclude: By induction, holds for all n ∈ ℕ. □
Strong Induction & Well-Ordering
Strong Induction: Assume P(1), P(2), ..., P(k) all true, prove P(k+1). Use when the (k+1) case depends on more than just the k case.

Well-Ordering Principle: Every non-empty subset of ℕ has a smallest element. Equivalent to induction. Used in proofs about ℤ (e.g. division algorithm, GCD existence).

In Real Analysis, the well-ordering principle underlies the Archimedean property of ℝ: ∀x ∈ ℝ, ∃n ∈ ℕ, n > x. This is used in almost every ε-N proof.
